    Write the file with the Write tool. Final file contents (verbatim, including trailing newline):

    ```ruby
    # Be sure to restart your server when you modify this file.

    # Define an application-wide content security policy.
    # See the Securing Rails Applications Guide for more information:
    # https://guides.rubyonrails.org/security.html#content-security-policy-header

    Rails.application.configure do
      config.content_security_policy do |policy|
        policy.default_src :self
        policy.font_src    :self, :data
        policy.img_src     :self, :data
        policy.object_src  :none
        policy.script_src  :self
        policy.style_src   :self
      end

      # Generate a per-request nonce for script-src and style-src.
      # Importmap renders an inline <script type="importmap"> tag that requires
      # this nonce; Rails injects it automatically via javascript_importmap_tags.
      config.content_security_policy_nonce_generator = ->(request) { request.session.id.to_s }
      config.content_security_policy_nonce_directives = %w[script-src style-src]
    end
    ```

    - `content_security_policy_nonce_directives` includes both `script-src` and `style-src` so Rails injects the nonce into both `javascript_importmap_tags` (inline `<script type="importmap">`) and any inline `<style>` tag emitted by helpers

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client (browser) → Reverse proxy (Kamal 2) | TLS terminates at the proxy; downstream traffic to Puma is plaintext over the internal network |
| Reverse proxy → Rails (Puma) | Rails assumes upstream TLS via `assume_ssl`; trusts `X-Forwarded-Proto` |
| Rendered HTML → browser JS engine | CSP enforced on browser; nonce-gated inline scripts (importmap tag) are the only inline JS permitted |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information disclosure (network sniffing) | All non-/up HTTP traffic | mitigate | `config.force_ssl = true` issues 301 to HTTPS for all paths except `/up`; HSTS header set by Rails SSL middleware. Verified in Task 1 acceptance criteria. |
| T-01-02 | Spoofing (proxy-bypass HTTP request hitting Rails directly) | Puma listener | mitigate | `config.assume_ssl = true` makes Rails treat all requests as HTTPS based on `X-Forwarded-Proto`; combined with Kamal's network isolation, direct HTTP access is not exposed publicly. Documented dependency on Kamal network topology in CONTEXT D-02. |
| T-01-03 | Denial of service (health check fails because of HTTP→HTTPS redirect loop) | Kamal deploy probe hitting `/up` | mitigate | `ssl_options = { redirect: { exclude: ->(request) { request.path == "/up" } } }` exempts the health endpoint from forced HTTPS redirect. Verified in Task 1 acceptance criteria. |
| T-01-04 | Tampering (XSS via injected inline `<script>` tags) | Browser DOM | mitigate | CSP enforced (not report-only) with `script-src :self` plus per-request nonce. Inline scripts without the correct nonce are blocked by the browser. No `'unsafe-inline'`. Verified in Task 2 acceptance criteria. |
| T-01-05 | Tampering (CSS injection executing JS via `<style>` exfiltration patterns) | Browser DOM | mitigate | `style_src :self` plus nonce on `style-src` directive prevents inline `<style>` injection. No `'unsafe-inline'`. |
| T-01-06 | Tampering (data exfiltration via `<object>` / `<embed>` / Flash) | Browser DOM | mitigate | `object_src :none` blocks all plugin/object embeds. |
| T-01-07 | Misconfiguration (CSP nonce generator binds to session that does not exist) | Initializer boot | mitigate | `request.session.id.to_s` is the Rails 8.1 default and works with the project's cookie store. If sessions become non-cookie-backed, this needs revisit. Task 3 boot test catches any initializer-time exception. |
| T-01-08 | Misconfiguration (CSP unintentionally weakened by adding `:https` or `'unsafe-inline'` during future edits) | Future maintenance | accept | Detected by brakeman in CI (Task 3 baselines a clean scan); reviewer-level discipline + grep gates in this plan's acceptance criteria. Low residual risk given solo-developer workflow. |
| T-01-09 | Repudiation / Information disclosure (CSP violations not logged) | Browser → server | accept | No CSP `report_uri` configured. Per D-05 the app has no production traffic yet; adding a reporting endpoint is deferred. Re-evaluate when v2 user-generated content is introduced. |
| T-01-10 | Elevation of privilege (insecure-cookie attribute missing on session cookie) | Browser cookie jar | mitigate | `config.force_ssl = true` triggers Rails' `ActionDispatch::SSL` middleware which sets `secure: true` on session and CSRF cookies automatically. No additional config required. |

All high-severity threats (T-01-01 through T-01-07, T-01-10) are mitigated. T-01-08 and T-01-09 are accepted with documented rationale and revisit triggers.
</threat_model>
